Primes and Security

How the modern world is built on a seemingly insignificant math peculiarity.

Why are primes such a big deal in mathematics? It's quirky, but of what use is it really? I have fallen ignorant of the value that a seemingly child-counting mathematical property can hold. Primes are extensively used in cryptography, random number generation and key exchange. Trust among 5 billion people using the internet rely on the humble prime.

Consider the problem where a sender need to send a message to a recipient:

  • that no one else can read (encryption),
  • where the recipient can verify the sender is the sender (authentication),
  • and without prior exchanging of a secret key that can be intercepted.

There are two types of cryptography, symmetric where the same key is used for encryption and decryption, and asymmetric where each person has a key that is publicly accessible and a key that stays private/secret. Symmetric cryptography immediately raises the question of key exchange.

Key Exchange

Diffie-Hellman (DH) key exchange was published in 1967 as a method to exchange a secret key over an insecure channel. It uses the hardness of the discrete logarithm problem to calculate an irreversible representation of a secret key. The representation can be publicly accessible since the secret cannot be obtained from it.

\[A = g^a \bmod{n}\]

Person A's secret key is \(a\), \(g\) is a base (usually 2 or 5) and \(n\) is a large number that makes it hard to get back to \(a\) from \(A\). Person B does the same and gives \(B\) to person A to compute:

\[B^a \bmod{n} = g^{b a} \bmod{n} = g^{a b} \bmod{n}\]

Person B's can also get to the same shared key by using \(A\) and \(b\).

\[A^b \bmod{n} = g^{a b} \bmod{n}\]

Note that \(g^{a b} \bmod{n}\) was established without ever exposing secret keys \(a\) and \(b\) to anyone and no one else can calculate it with only \(g\), \(n\), \(A\) and \(B\). However, there is one very important piece neglected, and that is \(n\).

If just any number is picked for \(n\) there is a risk that it may be very easy to reverse the discrete logarithm problem.

For example, if \(n=15\) and \(g=2\), then \(g^{a b} \bmod{n}\) only falls within the set of \(\{1,2,4,8\}\) instead of the ideally large set of \(\{1,2,...,n-1\}\). The solution to protect against such vulnerability is to use a large prime number for \(n\).

Inventing RSA

With asymmetric cryptography each person has a public key and a private key. Anyone can see the public key but only the owner ever knows the private key. The sender can encrypt a message with the recipient's public key and only the recipient can decrypt the message with the matching private key. Encryption is established without exchanging any secret keys. Rivest, Shamir and Adleman published RSA in 1977 that uses primes for such a function. It relies on the mathematical property that multiplying two large primes is easy but factoring their product back into the original primes is hard.

Key Idea 1)

RSA starts by generating two large prime numbers, \(p\) and \(q\).

\[n = p q\]

\(n\) is a new prime number that can be publicly shared while \(p\) and \(q\) remain secret because of the irreversibility of the product. Thus, \(p\) and \(q\) can still be used to calculate additional cryptographic elements.

The RSA authors, however, will build on math discovered three centuries earlier. The initial version of Euler's theorem was first published in 1736 as a proof of Fermat's little theorem (published without 1640). I wonder if Euler and Fermat were also grilled for showing the usefulness of their work to get funding like today.

Key Idea 2)

With modulo arithmetic Euler's theorem:

\[a^{\varphi(n)} \equiv 1 \pmod{n}\]

shows that there is a power (Euler's Totient) one can raise a number to, resulting in 1. This property can be used to obtain the original value of an encrypted value.

As an example, take \(n=10\) and \(a=3\). If we increment \(b\) in \(a^b \pmod{n}\) from 1 to 10, we get the sequence 3, 9, 7, 1, 3, 9, 7, 1, 3, 9.

Key Idea 3)

From Euler's theorem we also get the idea of raising numbers to a power to decrypt it. And, while we're at it, why not also use a power to encrypt it?

So, what if the encrypted number, \(c\), is

\[c=m^e \pmod{n}\]

and

\[m=c^d \pmod{n}=m^{ed} \pmod{n}\]

where \(e\) and \(d\) are encryption and decryption exponents.

All we need to do now is to calculate \(e\) and so that

\[m^{ed} \equiv m \pmod{n}\]

Well, Euler's theorem has a one in so we can substitute the \(1^k\) at the lone \(m\) with

\[m^{ed} = m (m^{\varphi(n)})^k\]

where k is the amount of modulo cycles, and now

\[m^{ed} = m^{1+k\varphi(n)}\]

So if we take the exponents as an equation \(ed = 1+k\varphi(n)\) or \(ed-k\varphi(n) = 1\) so that

\[ed \equiv 1 \pmod{\varphi(n)}\]
